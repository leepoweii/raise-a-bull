# follow-up-tracker

透過 `gog tasks` CLI 管理 Google Tasks，追蹤待辦事項與他人承諾。

## 設定

讀取 `../../params.json` 的 `tasks.default_list_id` 作為 `{LIST_ID}`。

如果 `tasks.default_list_id` 未設定，先用 `gog tasks lists` 列出所有 task list，請使用者選擇要使用哪一個。

## 工具

所有指令透過 `gog tasks` CLI 執行（OpenClaw 內建 `gog` skill）。

---

## 操作

### 1. 列出待辦（LIST）

**觸發時機**：使用者說「看一下待辦」、「有哪些要追蹤的」、「我的 tasks」。

**指令**：

```bash
gog tasks list {LIST_ID}
```

**常用 flags**：

- `--max=N` — 限制回傳數量（預設 20，最多 100）
- `--all` — 取得全部頁面
- `--due-max=YYYY-MM-DDT23:59:59Z` — 篩選截止日期之前的任務
- `-j` — JSON 輸出（方便程式處理）

**回覆格式**：

用乾淨的編號清單呈現，包含截止日期：

```
你目前有 3 個待辦：

1. 寄報價單給王先生 — 截止：03/15
2. 確認場地預約 — 截止：03/18
3. 回覆國發會文件 — 無截止日
```

---

### 2. 新增待辦（ADD）

**觸發時機**：使用者說「幫我記住」、「追蹤這件事」、「提醒我」、「XXX 答應要...」。

**指令**：

```bash
gog tasks add {LIST_ID} --title "任務標題" [--due YYYY-MM-DD] [--notes "備註"]
```

**參數判斷**：

- 使用者提到明確日期 → 設定 `--due`
- 使用者提到「下週X」、「月底前」→ 換算成 `YYYY-MM-DD` 後設定 `--due`
- 使用者提到某人的承諾 → 在 `--notes` 記錄承諾者與承諾內容
- 沒有提到期限 → 不設 `--due`

**防重複**：新增前先 `gog tasks list {LIST_ID}` 檢查是否已有相同或高度相似的任務。如果有，告知使用者並詢問是否仍要新增。

**回覆格式**：

```
已新增待辦：
- 標題：寄報價單給王先生
- 截止：2026-03-15
- 備註：王先生在 3/12 會議中要求的

需要追蹤其他事項嗎？
```

---

### 3. 完成待辦（DONE）

**觸發時機**：使用者說「這件事完成了」、「done」、「搞定了」。

**指令**：

```bash
gog tasks done {LIST_ID} {TASK_ID}
```

**流程**：

1. 如果使用者沒有指定 task ID，先列出待辦讓使用者選擇
2. 使用者確認後，標記完成

**回覆格式**：

```
「寄報價單給王先生」已完成！
```

---

### 4. 更新待辦（UPDATE）

**觸發時機**：使用者說「改一下」、「更新」、「延期」、「加個備註」。

**指令**：

```bash
gog tasks update {LIST_ID} {TASK_ID} [--title "新標題"] [--notes "新備註"] [--due YYYY-MM-DD]
```

**流程**：

1. 如果使用者沒有指定 task ID，先列出待辦讓使用者選擇
2. 根據使用者需求更新對應欄位

**回覆格式**：

```
已更新「寄報價單給王先生」：
- 截止日改為：2026-03-20
```

---

### 5. 刪除待辦（DELETE）

**觸發時機**：使用者說「刪掉」、「不用追蹤了」、「取消這個」。

**指令**：

```bash
gog tasks delete {LIST_ID} {TASK_ID} -y
```

**流程**：

1. 如果使用者沒有指定 task ID，先列出待辦讓使用者選擇
2. 刪除前向使用者確認

**回覆格式**：

```
已刪除「寄報價單給王先生」。
```

## 使用情境範例

| 使用者說 | 動作 |
|---|---|
| 「幫我記住要追蹤這件事」 | ADD — 詢問細節後新增 |
| 「XXX 答應下週給我報告，幫我追蹤」 | ADD — title: 追蹤 XXX 的報告, due: 下週, notes: XXX 承諾提供 |
| 「看一下我有哪些待辦」 | LIST — 列出所有未完成任務 |
| 「這件事完成了」 | DONE — 確認哪一個後標記完成 |
| 「提醒我下週五之前要寄文件」 | ADD — title: 寄文件, due: 下週五 |
| 「把截止日延到月底」 | UPDATE — 更新 due date |

---

## 重要規則

- 一律使用繁體中文回覆
- 使用者提到期限時，一定要設定 `--due`
- 使用者提到他人承諾時，在 `--notes` 記錄承諾者與承諾內容
- 新增前檢查是否有重複任務
- `{LIST_ID}` 從 `../../params.json` 的 `tasks.default_list_id` 取得，不要硬編碼
- 完成任務時給正面回饋
