# weather-cwa

查詢金門縣天氣預報（資料來源：中央氣象署開放資料）。

**此 skill 優先於內建 weather skill。** 所有金門天氣相關問題都走這裡。

## 設定

讀取 `../../params.json` 的以下欄位：

| 欄位 | 說明 | 預設值 |
|---|---|---|
| `weather.default_location` | 預設查詢鄉鎮 | `金城鎮` |
| `weather.default_dataset` | 預設資料集 | `F-D0047-085`（2 天預報） |

## 使用方式

用 web_fetch 呼叫中央氣象署 API。環境變數 `CWA_API_KEY` 是授權碼。

### 查詢金門天氣

```
GET https://opendata.cwa.gov.tw/api/v1/rest/datastore/{dataset}?Authorization={CWA_API_KEY}&format=JSON&locationName={location}
```

- `F-D0047-085`：金門縣未來 2 天鄉鎮天氣預報
- `F-D0047-087`：金門縣未來 1 週鄉鎮天氣預報
- locationName 可選：金城鎮、金湖鎮、金沙鎮、金寧鄉、烈嶼鄉（小金門）
- **預設查 `params.json` 的 `weather.default_location`**（若無設定，預設金城鎮）

### 回應解析

回應結構（注意大寫）：`records.Locations[0].Location[0].WeatherElement`

每個 WeatherElement 的 ElementName 是中文：

| ElementName | 說明 |
|---|---|
| 天氣預報綜合描述 | **主要使用這個** — 已整理好的完整描述 |
| 天氣現象 | 如「晴」「多雲時陰」「陰有雨」 |
| 溫度 | °C |
| 3小時降雨機率 | % |
| 體感溫度 | °C |
| 相對濕度 | % |
| 風速 | m/s |
| 風向 | 如「東北風」 |

每個 WeatherElement 包含 `Time` 陣列，每筆有 `StartTime`、`EndTime`、`ElementValue`。

### 回覆原則

**優先使用「天氣預報綜合描述」作為主內容。**

例如 API 回傳：
`「晴。降雨機率0%。溫度攝氏13至15度。寒冷。東北風 平均風速2-3級(每秒5公尺)。相對濕度61至68%。」`

就自然地整理成：
`「今天金門天氣不錯，晴天，13-15°C，不會下雨。不過東北風有點涼，體感偏冷，提醒大家加件外套。」`

若「天氣預報綜合描述」缺值，再退回個別欄位（溫度、降雨機率、天氣現象、風速）組合回覆。

### 行動建議

根據天氣狀況與組織類型，給一句實用建議。以下為範例模式：

**餐飲 / 零售類：**
- 晴天：「天氣好，今天客人應該不少」
- 下雨：「今天有雨，可能客人少一點，備料可以少準備」
- 強風：「風很大，戶外區域注意安全」
- 寒冷：「很冷，可以推薦熱飲或暖身品項」

**活動 / 觀光類：**
- 晴天：「適合戶外活動，可以安排室外行程」
- 下雨：「建議準備室內備案或提醒參加者帶傘」
- 強風：「注意戶外設施固定，考慮調整活動安排」
- 寒冷：「提醒參加者注意保暖」

**一般用途：**
- 晴天：「天氣不錯，適合外出」
- 下雨：「記得帶傘」
- 強風：「外出注意風勢」
- 寒冷：「注意保暖」

根據你的角色設定與組織屬性，選擇合適的建議風格。

### 錯誤處理

- API timeout → 「氣象資料暫時無法取得，稍後再試」
- 401/403 → 「API 授權有問題，請聯絡管理員」
- 429 → 「查詢太頻繁，稍後再試」
- 空資料 → 「目前沒有金門的天氣資料」

### 重要規則

1. 環境變數 `CWA_API_KEY` 放在 query parameter `Authorization`
2. CWA 回應結構可能因版本微調，先檢查 key 是否存在再解析
3. 不要把整包 JSON 丟出來，只回覆對使用者有用的資訊
4. 用 web_fetch，不要用 exec
5. 啟動時讀取 `../../params.json` 取得預設地點與資料集設定
